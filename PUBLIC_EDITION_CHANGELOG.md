# ProjectMind v4.0 Public-Edition Changelog

## Major Research Corrections

- Repositioned generic code graphs, MCP, project memory, temporal graphs, impact
  analysis, test selection, provenance, policy, and attestations as prior art.
- Added direct comparison with Codebase-Memory, Repository Intelligence Graph,
  PROJECTMEM, KGCompass, Prometheus, RepoDoc, Zep/Graphiti, CodePlan, Codified
  Context, and assertion-based test selection.
- Narrowed the contribution to a commit-DAG continuity-control chain.

## New Core Mechanisms

- Commit-DAG Evidence Frontier and canonical digest.
- Five-dimensional claim-quality vector.
- Append-only Authority Promotion Ledger.
- Immutable Change Consequence Contract.
- Predicted-versus-observed graph/test/policy delta reconciliation.
- Context Receipts and Continuity Receipts.
- Monotonic-caution rule when evidence weakens.

## Architecture Repairs

- Replaced linear commit validity assumptions with ancestry/reachability rules.
- Separated entities from claims and added many-to-many claim/evidence links.
- Separated signature authenticity from factual confidence and policy authority.
- Made Git-native operation the primary MVP and external controllers optional.
- Added explicit context omissions and retrieval limits.
- Added branch-confusion, authority-laundering, stale-replay, prediction-laundering,
  and receipt-forgery threats.
- Added SQLite patched-version requirements based on the documented 2026 WAL fix.
- Reclassified the seven-day promise as a ten-day feasibility spike.

## Publication Repairs

- Removed internal workspace numbering and private pack bindings.
- Standardized the document as v4.0 Public Research Edition.
- Added abstract, keywords, author record, citation, disclosure, rights, current
  references, novelty boundary, falsifiability, and publication limitations.
- Preserved implementation detail while removing unsupported firstness,
  readiness, validation, patent, and compliance implications.
